= A2A Protocol Features in Apicurio Registry

Apicurio Registry implements the link:https://a2a-protocol.org/latest/specification/[A2A Protocol v1.0], enabling AI agent discovery, registration, and lifecycle management.

== Overview

The A2A protocol enables AI agents to discover and communicate with each other through standardized endpoints. Apicurio Registry serves as both:

1. **An A2A Agent** - Exposing its own capabilities (schema validation, search, artifact management)
2. **An Agent Registry** - Storing and discovering other agents' Agent Cards with entitlement-based access control

|`apicurio.a2a.agent.protocol-version`
|`1.0`

|`apicurio.a2a.agent.documentation-url`
|_none_
|URL to agent documentation

|`apicurio.a2a.agent.icon-url`
|_none_
|URL to agent icon

|`apicurio.a2a.agent.provider.organization`
|`Apicurio`
|Provider organization name

|`apicurio.a2a.agent.provider.url`

[cols="2,1,3"]
|===
|Property |Default |Description

== Discovery Endpoints

Apicurio Registry provides a three-tier agent discovery model aligned with the link:https://github.com/a2aproject/A2A/discussions/741[A2A community proposal].

=== GET /.well-known/a2a

Returns the Agent Card for this Apicurio Registry instance (canonical A2A v1.0 path). The legacy path `/.well-known/agent.json` is also supported.

**Authentication:** None required

**Response:**

[source,json]
----
{
  "name": "Apicurio Registry",
  "description": "API and Schema Registry with A2A Agent support",
  "version": "3.0.0",
  "protocolVersion": "1.0",
  "supportedInterfaces": [
    {
      "url": "http://localhost:8080",
      "protocolBinding": "http+json",
      "protocolVersion": "1.0"
    }
  ],
  "provider": {
    "organization": "Apicurio",
    "url": "https://www.apicur.io"
  },
  "capabilities": {
    "streaming": false,
    "pushNotifications": false,
    "extendedAgentCard": false
  },
  "skills": [
    { "id": "schema-validation", "name": "Schema Validation", "description": "...", "tags": ["schema"] },
    { "id": "agent-discovery", "name": "Agent Discovery", "description": "...", "tags": ["a2a"] }
  ],
  "defaultInputModes": ["text"],
  "defaultOutputModes": ["text"],
  "securitySchemes": {
    "bearer": { "type": "httpAuth", "scheme": "Bearer" }
  }
}
----

=== GET /.well-known/agents/public

Returns agents explicitly marked as public. No authentication required.

Only Agent Card artifacts labeled with `apicurio.agent.visibility=public` are returned.

**Authentication:** None required

**Query Parameters:**

[cols="1,1,3"]
|===
|Parameter |Type |Description
|`offset` |integer |Pagination offset (default: 0)
|`limit` |integer |Pagination limit (default: 20)
|===

**Example:**
[source,bash]
----
curl "http://localhost:8080/.well-known/agents/public"
----

=== GET /.well-known/agents/entitled

Returns agents the authenticated caller is entitled to access, filtered by visibility:

- **public** agents are always included
- **entitled** agents (default) are included for any authenticated user
- **private** agents are included only for the artifact owner or admins

**Authentication:** Required (bearer token)

**Query Parameters:**

[cols="1,1,3"]
|===
|Parameter |Type |Description
|`offset` |integer |Pagination offset (default: 0)
|`limit` |integer |Pagination limit (default: 20)
|===

**Example:**
[source,bash]
----
curl -H "Authorization: Bearer <token>" \
  "http://localhost:8080/.well-known/agents/entitled"
----

=== POST /.well-known/agents/search

Advanced agent search with combined text query and structured filters. Respects the caller's entitlements when entitlement filtering is enabled.

**Authentication:** Required (bearer token)

**Request Body:**

[source,json]
----
{
  "query": "customer support",
  "filters": {
    "capabilities": { "streaming": true },
    "skills": ["ticket-creation"],
    "labels": { "department": "engineering" },
    "inputModes": ["text"],
    "outputModes": ["text"],
    "protocolBindings": ["http+json"]
  },
  "limit": 20,
  "offset": 0
}
----

All filter fields are optional. When multiple filters are specified, results must match all of them.

**Example:**
[source,bash]
----
curl -X POST "http://localhost:8080/.well-known/agents/search" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <token>" \
  -d '{"query": "translation", "filters": {"skills": ["translate"]}, "limit": 10}'
----

=== GET /.well-known/agents

Search registered Agent Cards with query parameter-based filtering. This is the original search endpoint, maintained for backward compatibility.

**Authentication:** Read-level (or anonymous if enabled)

**Query Parameters:**

[cols="1,1,3"]
|===
|Parameter |Type |Description
|`name` |string |Filter by agent name (partial match)
|`skill` |string (repeatable) |Filter by skill ID
|`capability` |string |Filter by capability (e.g., `streaming:true`)
|`inputMode` |string |Filter by input mode
|`outputMode` |string |Filter by output mode
|`offset` |integer |Pagination offset (default: 0)
|`limit` |integer |Pagination limit (default: 20)
|===

=== GET /.well-known/agents/\{groupId}/\{artifactId}

Retrieve a specific registered Agent Card by group and artifact ID.

**Authentication:** Read-level

**Query Parameters:**

- `version` (optional) - Specific version expression (defaults to latest)

== Agent Visibility Model

Agent Cards support three visibility levels, controlled via the `apicurio.agent.visibility` artifact label:

[cols="1,3"]
|===
|Visibility |Behavior

|`public`
|Visible to everyone, including unauthenticated clients via `/.well-known/agents/public`

|`entitled`
|Visible to any authenticated user (this is the default)

|`private`
|Visible only to the artifact owner and registry admins
|===

=== Setting Visibility

Set the visibility label when creating or updating an Agent Card:

[source,bash]
----
# Set an agent as public
curl -X PUT "http://localhost:8080/apis/registry/v3/groups/my-agents/artifacts/my-agent" \
  -H "Content-Type: application/json" \
  -d '{"labels": {"apicurio.agent.visibility": "public"}}'
----

=== How Entitlements Work

Apicurio Registry maps the link:https://github.com/a2aproject/A2A/discussions/741[A2A community proposal]'s entitlement concept to its existing RBAC model:

- **Groups** act as organizational namespaces
- **"Entitling" a client** = granting read access to the group(s) containing the Agent Card artifacts
- **Client authentication** is delegated to the external IdP (Keycloak, OIDC)
- No new authorization primitives are needed

[cols="1,1,2"]
|===
|Community Role |Apicurio Role |Notes

|Administrator |`sr-admin` |Full control over all agents
|Catalog Manager |`sr-developer` |Can create/update/delete Agent Cards
|User |`sr-readonly` |Can browse and discover agents
|===

== Agent Card Schema (v1.0)

Agent Cards conform to the A2A Protocol v1.0 specification. The required fields are:

- `name` — Agent name (non-empty string)
- `description` — What the agent does
- `version` — Agent version
- `supportedInterfaces` — Array of protocol interfaces (at least one)
- `capabilities` — Agent capabilities object
- `skills` — Array of skills (at least one, each with `id`, `name`, `description`, `tags`)
- `defaultInputModes` — Supported input MIME types
- `defaultOutputModes` — Supported output MIME types

Optional fields include `protocolVersion`, `provider`, `securitySchemes`, `securityRequirements`, `iconUrl`, `documentationUrl`, and `signatures`.

=== Creating an Agent Card

[source,bash]
----
curl -X POST "http://localhost:8080/apis/registry/v3/groups/ai-agents/artifacts" \
  -H "Content-Type: application/json" \
  -d '{
    "artifactId": "my-agent",
    "artifactType": "AGENT_CARD",
    "firstVersion": {
      "version": "1.0.0",
      "content": {
        "contentType": "application/json",
        "content": "{\"name\": \"My Agent\", \"description\": \"An example agent\", \"version\": \"1.0.0\", \"supportedInterfaces\": [{\"url\": \"https://my-agent.example.com\", \"protocolBinding\": \"http+json\", \"protocolVersion\": \"1.0\"}], \"capabilities\": {}, \"skills\": [{\"id\": \"task\", \"name\": \"Task Execution\", \"description\": \"Execute tasks\", \"tags\": [\"general\"]}], \"defaultInputModes\": [\"text\"], \"defaultOutputModes\": [\"text\"]}"
      }
    }
  }'
----

== Compatibility Rules

When versioning Agent Cards, compatibility rules are enforced:

**Compatible Changes (allowed):**

- Adding new skills
- Adding new capabilities
- Adding security schemes
- Adding input/output modes
- Adding supported interfaces

**Incompatible Changes (breaking):**

- Removing skills
- Removing or disabling capabilities
- Removing supported interfaces
- Changing protocol version on an existing interface
- Removing security schemes
- Removing input/output modes

Apicurio Registry exposes itself as an A2A agent with these skills:

[cols="1,2"]
|===
|Skill ID |Description

|`schema-validation`
|Validate schemas against format specifications (Avro, JSON Schema, Protobuf, OpenAPI, etc.)

|`schema-search`

== Troubleshooting

=== A2A Endpoints Return 404

Ensure A2A is enabled:
[source,properties]
----
apicurio.a2a.enabled=true
----

=== Public Discovery Endpoint Returns 404

Ensure public discovery is enabled (it is by default):
[source,properties]
----
apicurio.a2a.public-discovery.enabled=true
----

=== Agent Card Validation Fails

Check that all v1.0 required fields are present:

1. `name`, `description`, `version` — non-empty strings
2. `supportedInterfaces` — non-empty array with `url`, `protocolBinding`, `protocolVersion`
3. `capabilities` — object (can be empty)
4. `skills` — non-empty array, each skill must have `id`, `name`, `description`, `tags`
5. `defaultInputModes`, `defaultOutputModes` — non-empty string arrays

=== Private Agents Visible to Other Users

Check that the `apicurio.agent.visibility` label is set to `private` on the artifact:
[source,bash]
----
curl "http://localhost:8080/apis/registry/v3/groups/{groupId}/artifacts/{artifactId}" \
  | jq '.labels'
----
